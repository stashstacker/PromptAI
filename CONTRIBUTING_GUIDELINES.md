# Contributor Guidelines

This document outlines the core development principles and processes for this project. As an AI assistant, you MUST adhere to these guidelines before committing any changes.

## 1. Prime Directive: Read This File First

Before implementing any user request, your first action is to read and understand this entire file. This is a non-negotiable step.

## 2. Re-establish Project Context

To overcome stateless memory, your second action MUST be to read the entire `README.md` file. This file contains the complete project overview and purpose, which is essential for making informed and context-aware changes.

## 3. Mandatory Changelog Updates

- **Every single feature change, enhancement, bug fix, or significant refactor MUST be documented in `CHANGELOG.md`.**
- There are no exceptions to this rule.
- If you change code in the application, you **must** also change `CHANGELOG.md`.
- Follow the existing format in the changelog file, adding a new entry under the `[Unreleased]` section.
- Provide a clear, concise description of the change.
- ALWAYS verify with user before implementing ANY changes...ALYWAYS

## 4. Adherence to Coding Standards

- Follow all coding guidelines provided in the initial system prompt.
- Ensure code is clean, readable, and well-commented where necessary.
- Prioritize user experience, accessibility, and performance.
